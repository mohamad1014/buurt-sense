# Agent Guidelines

- Prefer Python 3.11 features with full type hints.
- Keep dependencies minimal and document new commands in README updates when relevant.
- Format Python code with Black defaults.
- Try to keep code to minimal changes unless necessary.
- Use docstrings for all functions and classes.
- Write unit tests for new functionality and ensure existing tests pass.
- Follow PEP 8 style guidelines.
- Use descriptive variable and function names.
- Comment complex logic for clarity.
- Ensure code is modular and reusable.

- Try to add code that is actually useful for users.
- Avoid adding unnecessary complexity or bloat.
- Focus on maintainability and readability.
- Review code for potential performance improvements.

- Use available mcp servers to test functionalities
- Add tests to the test suite when needed
- Apply the user instructions.
- Keep in mind that you are running on Ubuntu within WSL.
